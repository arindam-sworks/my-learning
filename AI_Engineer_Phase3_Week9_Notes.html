01

The ReAct Pattern

ReAct (Reasoning + Acting) is a 2022 paper from Google that showed a simple but powerful idea: letting the LLM interleave reasoning steps with actions dramatically improves performance on tasks that require external information. The model thinks, acts, observes the result, thinks again — looping until it has an answer.

Thought → Action → Observation → Thought → Action → Observation → … → Final Answer

A key thing to understand: the "thinking" is not magic. The model generates text that looks like reasoning, and that text becomes context for the next token prediction. More reasoning tokens = better grounding for the final answer. The loop closes because tool results are appended to the message history, so the model always has full context of what it has tried and learned.

A real ReAct trace

Here is what the model actually produces for the question "How many seconds has Elon Musk been alive?":

thoughtI need to find Elon Musk's birth date to calculate his age in seconds. Let me search for it. actionweb_search("Elon Musk date of birth") obsElon Musk was born on June 28, 1971. thoughtNow I need to calculate the number of seconds from June 28, 1971 to today's date. I'll use the calculator tool. actioncalculator("(2025 - 1971) * 365.25 * 24 * 3600 + (days_since_june28_2025 * 86400)") obsResult: 1,701,388,800 (approximate, assuming today is mid-2025) final answerElon Musk has been alive for approximately 1.7 billion seconds (born June 28, 1971). The exact number depends on today's date.

🧠 Why ReAct outperforms single-shot prompting

Without tools, the model would guess Elon Musk's birth year from training data — possibly wrong or outdated — and do arithmetic in its head — often wrong. With ReAct: the search tool guarantees the correct birth date. The calculator tool guarantees correct arithmetic. The model's job is just to orchestrate — decide what to search, what to compute, and when it has enough information to answer. This division of labor — LLM for reasoning, tools for execution — is the architecture of every production agent.

Anthropic's Native Tool Use API

Anthropic's API has first-class support for tool use. You define tools as JSON schemas, pass them with the request, and the model returns either a text block (thinking/answer) or a tool_use block (a structured tool call). No prompt parsing required.

Python — Tool use API basics
import anthropic, json
client = anthropic.Anthropic()

# ── Define tools ────────────────────────────────────────────────
tools = [
    {
        "name": "web_search",
        "description": "Search the web for current information. Use for facts, dates, recent events.",
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "The search query"
                }
            },
            "required": ["query"]
        }
    }
]

# ── First call — model decides to use a tool ────────────────────
response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[{"role": "user", "content": "When was Elon Musk born?"}]
)

print(response.stop_reason)  # "tool_use" — model wants to call a tool
print(response.content)
# [
#   TextBlock(text="I'll search for that.", type="text"),
#   ToolUseBlock(id="toolu_abc123", name="web_search",
#                input={"query": "Elon Musk date of birth"}, type="tool_use")
# ]

# ── Extract the tool call ────────────────────────────────────────
tool_block = next(b for b in response.content if b.type == "tool_use")
tool_name   = tool_block.name    # "web_search"
tool_input  = tool_block.input   # {"query": "Elon Musk date of birth"}
tool_use_id = tool_block.id      # "toolu_abc123" — needed for response

# ── Execute the tool ────────────────────────────────────────────
tool_result = "Elon Musk was born on June 28, 1971."   # your real implementation

# ── Return the result to the model ──────────────────────────────
# CRITICAL: the full conversation history must be preserved
follow_up = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    tools=tools,
    messages=[
        {"role": "user",    "content": "When was Elon Musk born?"},
        {"role": "assistant", "content": response.content},  # full assistant response
        {"role": "user",    "content": [{                     # tool result as user turn
            "type":        "tool_result",
            "tool_use_id": tool_use_id,
            "content":     tool_result,
        }]}
    ]
)

print(follow_up.stop_reason)   # "end_turn" — model is done
print(follow_up.content[0].text)  # "Elon Musk was born on June 28, 1971."
The message structure rule

Tool results are sent as user-role messages with type tool_result, not as assistant messages. The tool_use_id must match the ID from the assistant's tool_use block exactly. If you pass multiple tool calls in one turn, you must return all their results together in one user message before the model can continue.

Tool Definitions & Dispatch

A well-designed tool definition is half the work of building a reliable agent. The model uses the description to decide when to call the tool, and the input_schema to know what arguments to pass. Vague descriptions produce incorrect tool choices.

Python — Tool registry pattern
import math, re
from typing import Callable, Any
from dataclasses import dataclass

@dataclass
class Tool:
    name:        str
    description: str
    schema:      dict           # JSON Schema for input_schema
    fn:          Callable       # actual Python function

    def to_api_format(self) -> dict:
        """Convert to Anthropic API tool definition."""
        return {
            "name":         self.name,
            "description":  self.description,
            "input_schema": self.schema,
        }

    def execute(self, **kwargs) -> str:
        """Execute the tool and always return a string."""
        try:
            result = self.fn(**kwargs)
            return str(result)
        except Exception as e:
            return f"ERROR: {type(e).__name__}: {e}"

# ── Tool implementations ─────────────────────────────────────────
def _calculator(expression: str) -> str:
    """Safe eval: only allows numbers, operators, math functions."""
    allowed = set("0123456789+-*/().,% \n")
    # Allow math.* functions
    cleaned = re.sub(r'math\.\w+', lambda m: m.group(), expression)
    if not all(c in allowed for c in re.sub(r'math\.\w+', '', cleaned)):
        raise ValueError("Unsafe expression")
    result = eval(expression, {"__builtins__": {}, "math": math})
    return f"{result:,.6g}"

def _web_search(query: str) -> str:
    """Real web search using DuckDuckGo (free, no API key)."""
    from duckduckgo_search import DDGS
    with DDGS() as ddgs:
        results = list(ddgs.text(query, max_results=3))
    if not results:
        return "No results found."
    parts = [f"[{r['title']}]\n{r['body']}" for r in results]
    return "\n\n".join(parts)

def _get_current_date() -> str:
    from datetime import date
    return f"Today is {date.today().isoformat()}"

# ── Tool registry ────────────────────────────────────────────────
TOOLS = [

            "Evaluate a mathematical expression. Use for arithmetic, unit conversion, "

        ),
        schema={
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "A valid Python math expression to evaluate"
                }
            },
            "required": ["expression"]
        },
        fn=_calculator
    ),
    Tool(
        name="web_search",
        description=(
            "Search the web for current facts, people, events, and data. "

            "Returns top 3 results with titles and summaries."
        ),
        schema={
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",

TOOL_MAP = {t.name: t for t in TOOLS}
Writing tool descriptions that work

The model uses your description to decide when to call the tool. Be explicit about: what it does, when to use it, and what format the input should be. Include an example input for complex schemas. Vague descriptions like "a tool for computation" produce wrong choices — be specific: "evaluate a Python math expression, e.g. (2025-1971)*365.25*24*3600."

The Agent Loop

The agent loop is the engine of a ReAct agent. It runs until the model produces a final answer (stop_reason == "end_turn") or hits a safety limit. Understanding the message accumulation pattern — how the history grows with each step — is essential.

Python — Core agent loop (without error handling yet)
def run_agent_loop(
    user_query:  str,
    tools:       list[Tool],
    system:      str = "",
    max_steps:   int = 10,
) -> dict:
    tool_defs = [t.to_api_format() for t in tools]
    tool_map  = {t.name: t for t in tools}

    # ── History accumulates every turn ───────────────────────────
    messages = [{"role": "user", "content": user_query}]
    steps    = []

    for step in range(max_steps):
        # Call the model with full history + tool definitions
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=4096,
            system=system,
            tools=tool_defs,
            messages=messages,
        )

        # Add assistant response to history
        messages.append({"role": "assistant", "content": response.content})

        # ── STOPPING CONDITIONS ───────────────────────────────────
        if response.stop_reason == "end_turn":
            final_text = next(
                (b.text for b in response.content if b.type == "text"), ""
            )
            return {"answer": final_text, "steps": steps, "total_steps": step+1}

        if response.stop_reason != "tool_use":
            break   # unexpected stop reason

        # ── TOOL EXECUTION ────────────────────────────────────────
        tool_results = []
        for block in response.content:
            if block.type != "tool_use": continue

            tool   = tool_map.get(block.name)
            result = tool.execute(**block.input) if tool else f"Unknown tool: {block.name}"

            steps.append({
                "step":       step + 1,
                "tool":       block.name,
                "input":      block.input,
                "output":     result,
            })

            tool_results.append({
                "type":        "tool_result",
                "tool_use_id": block.id,
                "content":     result,
            })

        # ── Append all tool results as one user turn ─────────────
        messages.append({"role": "user", "content": tool_results})

    return {"answer": "Max steps reached without final answer.",
            "steps": steps, "total_steps": max_steps}

Message history growth — what the model sees

Text — Message history after 2 tool calls
# After step 1 (search): messages contains 3 items
messages = [
    {"role": "user",      "content": "How many seconds has Elon Musk been alive?"},
    {"role": "assistant", "content": [TextBlock("I'll search..."),
                                            ToolUseBlock("web_search", {"query":...})]},
    {"role": "user",      "content": [{"type":"tool_result", "content": "Born June 28 1971"}]},
]

# After step 2 (calculator): messages contains 5 items
# ...and so on — the ENTIRE history is sent every call.
# Token cost grows with each step. A 10-step agent on sonnet
# can easily consume 50,000+ input tokens across all calls.
# This is why max_steps and early stopping matter for cost control.
Why history grows and why it matters

Every API call sends the complete message history. A 10-step agent with 500-token steps sends roughly 1+2+3+…+10 = 55 "step-loads" of tokens across all calls. At Sonnet pricing, a complex 15-step agent can cost $0.10–$0.30. Always track cost per agent run with a budget guard, and set max_steps aggressively — most tasks should complete in 3–5 steps.

Stopping Conditions

A runaway agent burns money and can loop indefinitely. You need multiple independent stopping conditions — never rely on just one.

Python — Complete stopping condition system
import time
from dataclasses import dataclass, field
from typing import Optional

@dataclass
class AgentConfig:
    max_steps:          int   = 15      # hard cap on tool calls
    max_time_seconds:   float = 60.0   # wall-clock timeout
    max_cost_usd:       float = 0.50   # spend budget per run
    max_input_tokens:   int   = 100_000 # context size guard
    same_tool_repeat:   int   = 3      # max identical calls in a row

@dataclass
class AgentState:
    start_time:  float        = field(default_factory=time.time)
    total_input_tokens:  int  = 0
    total_output_tokens: int  = 0
    last_tool_calls: list     = field(default_factory=list)  # for loop detection

    @property
    def elapsed(self) -> float:
        return time.time() - self.start_time

    @property
    def cost_usd(self) -> float:
        return (self.total_input_tokens  * 3.0 +
                self.total_output_tokens * 15.0) / 1_000_000

class StopCondition(Exception):
    def __init__(self, reason: str):
        self.reason = reason

def check_stopping_conditions(
    step:     int,
    state:    AgentState,
    config:   AgentConfig,
    new_tool_call: Optional[str] = None,
) -> None:
    """Raise StopCondition if any limit is exceeded."""

    if step >= config.max_steps:
        raise StopCondition(f"Max steps ({config.max_steps}) reached")

    if state.elapsed > config.max_time_seconds:
        raise StopCondition(f"Timeout ({state.elapsed:.1f}s > {config.max_time_seconds}s)")

    if state.cost_usd > config.max_cost_usd:
        raise StopCondition(f"Budget exceeded (${state.cost_usd:.4f} > ${config.max_cost_usd})")

    if state.total_input_tokens > config.max_input_tokens:
        raise StopCondition(f"Context limit ({state.total_input_tokens} tokens)")

    # Loop detection: same tool called with same args N times
    if new_tool_call:
        state.last_tool_calls.append(new_tool_call)
        if len(state.last_tool_calls) >= config.same_tool_repeat:
            recent = state.last_tool_calls[-config.same_tool_repeat:]
            if len(set(recent)) == 1:   # all identical
                raise StopCondition(f"Loop detected: '{new_tool_call}' repeated {config.same_tool_repeat} times")

max_stepsHard cap on tool calls — prevents infinite loops10–15 for most tasks

Error Handling in Agent Loops

Tool errors should not crash the agent. The correct pattern: catch all tool exceptions, return the error message as the tool result, and let the model decide how to recover. Models handle errors surprisingly well when given clear error descriptions.

Python — Resilient agent loop with full error handling
from anthropic import RateLimitError, APIStatusError
import time, json

def run_agent(
    query:    str,
    tools:    list[Tool],
    system:   str = "",
    config:   AgentConfig = None,
) -> dict:
    config   = config or AgentConfig()
    state    = AgentState()
    tool_map = {t.name: t for t in tools}
    messages = [{"role": "user", "content": query}]
    trace    = []

    try:
        for step in range(config.max_steps):
            check_stopping_conditions(step, state, config)

            # ── API call with retry on rate limit ────────────────
            for attempt in range(3):
                try:
                    resp = client.messages.create(
                        model="claude-sonnet-4-20250514",
                        max_tokens=4096,
                        system=system,
                        tools=[t.to_api_format() for t in tools],
                        messages=messages,
                    )
                    break
                except RateLimitError:
                    if attempt == 2: raise
                    time.sleep(2 ** attempt)

            # Track token usage
            state.total_input_tokens  += resp.usage.input_tokens
            state.total_output_tokens += resp.usage.output_tokens
            check_stopping_conditions(step, state, config)   # re-check after token update

            messages.append({"role": "assistant", "content": resp.content})

            # ── Done ─────────────────────────────────────────────
            if resp.stop_reason == "end_turn":
                answer = next((b.text for b in resp.content if b.type == "text"), "")
                return _build_result(answer, trace, state, "success")

            # ── Execute tools ─────────────────────────────────────
            tool_results = []
            for block in resp.content:
                if block.type != "tool_use": continue

                check_stopping_conditions(step, state, config,
                                          new_tool_call=f"{block.name}:{json.dumps(block.input, sort_keys=True)}")

                tool = tool_map.get(block.name)
                if tool:
                    result = tool.execute(**block.input)   # errors caught inside execute()
                else:
                    result = f"ERROR: Unknown tool '{block.name}'. Available: {list(tool_map)}"

                trace.append({"step":step+1,"tool":block.name,"input":block.input,"output":result})
                tool_results.append({
                    "type":        "tool_result",
                    "tool_use_id": block.id,
                    "content":     result,
                    # Mark errors so model knows the tool failed
                    "is_error":    result.startswith("ERROR:"),
                })

            messages.append({"role": "user", "content": tool_results})

    except StopCondition as e:
        return _build_result(f"Stopped: {e.reason}", trace, state, e.reason)
    except Exception as e:
        return _build_result(f"Fatal error: {e}", trace, state, "error")

    return _build_result("Max steps reached.", trace, state, "max_steps")

def _build_result(answer, trace, state, stop_reason) -> dict:
    return {
        "answer":       answer,
        "trace":        trace,
        "steps_used":   len(trace),
        "stop_reason":  stop_reason,
        "cost_usd":     round(state.cost_usd, 5),
        "elapsed_s":    round(state.elapsed, 2),
        "total_tokens": state.total_input_tokens + state.total_output_tokens,
    }
Pass errors back to the model — don't re-raise

When a tool fails, return the error as the tool result with "is_error": True. The model will read the error message, reason about it, and often try a different approach — a different query, a corrected expression, or a graceful acknowledgment that the tool can't help. Re-raising the exception kills the agent entirely. Error messages are just observations in the loop.

Logging Agent Reasoning Traces

Agents are hard to debug because failures are non-deterministic. A trace that captures every thought, action, and observation is your only debugging tool. Without traces, a failing agent is a black box.

Python — Structured trace logger
import json, logging
from datetime import datetime
from pathlib import Path

class AgentTracer:
    def __init__(self, run_id: str, log_dir: str = "./traces"):
        self.run_id  = run_id
        self.log_dir = Path(log_dir)
        self.log_dir.mkdir(exist_ok=True)
        self.events  = []
        self.start   = datetime.utcnow()

        # Also log to console for live debugging
        logging.basicConfig(level=logging.INFO,
                            format='%(asctime)s | %(levelname)s | %(message)s')
        self.log = logging.getLogger(run_id)

    def log_thought(self, step: int, text: str):
        self._add({"type":"thought","step":step,"text":text})
        self.log.info(f"Step {step} [THOUGHT] {text[:120]}...")

    def log_tool_call(self, step: int, name: str, inp: dict):
        self._add({"type":"action","step":step,"tool":name,"input":inp})
        self.log.info(f"Step {step} [ACTION]  {name}({json.dumps(inp)[:80]})")

    def log_observation(self, step: int, result: str, is_error=False):
        self._add({"type":"observation","step":step,"result":result,"is_error":is_error})
        level = logging.ERROR if is_error else logging.INFO
        self.log.log(level, f"Step {step} [OBS]     {result[:120]}")

    def log_final(self, answer: str, metadata: dict):
        self._add({"type":"final","answer":answer,**metadata})
        self.log.info(f"FINAL ANSWER: {answer[:120]}")
        self.log.info(f"Cost: ${metadata.get('cost_usd',0):.5f} | Steps: {metadata.get('steps_used',0)}")

    def save(self):
        path = self.log_dir / f"{self.run_id}.json"
        path.write_text(json.dumps({
            "run_id":    self.run_id,
            "started_at": self.start.isoformat(),
            "events":    self.events,
        }, indent=2))
        print(f"Trace saved to {path}")

    def _add(self, event: dict):
        event["ts"] = (datetime.utcnow() - self.start).total_seconds()
        self.events.append(event)

# ── Integrate tracer into agent loop ─────────────────────────────
import uuid

def run_traced_agent(query: str, tools: list[Tool]) -> dict:
    run_id = str(uuid.uuid4())[:8]
    tracer = AgentTracer(run_id)

    # Inside the loop, after each block:
    for block in resp.content:
        if block.type == "text":
            tracer.log_thought(step, block.text)
        elif block.type == "tool_use":
            tracer.log_tool_call(step, block.name, block.input)
            result = tool_map[block.name].execute(**block.input)
            is_err = result.startswith("ERROR:")
            tracer.log_observation(step, result, is_error=is_err)

    result = run_agent(query, tools)
    tracer.log_final(result["answer"], result)
    tracer.save()
    return result
What to capture in every trace

Minimum viable trace: timestamp, step number, event type (thought/action/observation/final), content. Add: token count per step, cumulative cost, tool latency. When debugging a failing agent, the trace tells you exactly where the reasoning went wrong — whether the model misunderstood the task, called the wrong tool, or got stuck in a loop.

Full ReAct Agent — Complete Implementation

All components assembled: tool registry, agent loop, stopping conditions, error handling, and trace logging in one clean module.

Python — react_agent.py (complete, runnable)
"""
react_agent.py — ReAct agent in pure Python, no frameworks.
pip install anthropic duckduckgo-search
"""
import anthropic, json, math, re, time, uuid, logging
from dataclasses import dataclass, field
from typing import Callable, Optional
from datetime import date

logging.basicConfig(level=logging.INFO, format='%(levelname)s | %(message)s')
log    = logging.getLogger("agent")
client = anthropic.Anthropic()

# ═══════════════════════════════════════
#  TOOLS
# ═══════════════════════════════════════
@dataclass
class Tool:
    name: str; description: str; schema: dict; fn: Callable
    def api(self): return {"name":self.name,"description":self.description,"input_schema":self.schema}
    def run(self,**kw):
        try: return str(self.fn(**kw))
        except Exception as e: return f"ERROR: {e}"

def _calc(expression: str) -> str:
    safe = all(c in "0123456789+-*/().,% \nmath." or c.isalpha() for c in expression)
    if not safe: raise ValueError("Unsafe expression")
    r = eval(expression, {"__builtins__":{}, "math":math})
    return f"{r:,.6g}"

def _search(query: str) -> str:
    from duckduckgo_search import DDGS
    with DDGS() as d: r = list(d.text(query, max_results=3))
    return "\n\n".join(f"[{x['title']}]\n{x['body']}" for x in r) if r else "No results."

def _today() -> str: return f"Today is {date.today().isoformat()}"

TOOLS = [
    Tool("calculator",
         "Evaluate Python math expression. E.g. (2025-1971)*365.25*24*3600",
         {"type":"object","properties":{"expression":{"type":"string"}},"required":["expression"]},
         _calc),
    Tool("web_search",
         "Search web for current facts. Returns top 3 results.",
         {"type":"object","properties":{"query":{"type":"string"}},"required":["query"]},
         _search),
    Tool("get_today",
         "Returns today's date. Call before any date arithmetic.",
         {"type":"object","properties":{},"required":[]},
         _today),
]

# ═══════════════════════════════════════
#  AGENT LOOP
# ═══════════════════════════════════════
SYSTEM = """You are a helpful assistant with access to tools.
Think step by step. Use tools when you need current information or calculation.
When you have enough information, give a direct final answer.
Do not call the same tool with identical arguments more than twice."""

def run(query: str, max_steps: int = 10, max_cost: float = 0.50) -> dict:
    run_id   = str(uuid.uuid4())[:8]
    tmap     = {t.name: t for t in TOOLS}
    msgs     = [{"role":"user","content":query}]
    trace    = []
    cost     = 0.0
    t0       = time.time()

    log.info(f"[{run_id}] START: {query}")

    for step in range(max_steps):
        if cost > max_cost:
            break

        resp = client.messages.create(
            model="claude-sonnet-4-20250514", max_tokens=4096,
            system=SYSTEM, tools=[t.api() for t in TOOLS], messages=msgs
        )
        cost += (resp.usage.input_tokens*3+resp.usage.output_tokens*15)/1e6
        msgs.append({"role":"assistant","content":resp.content})

        if resp.stop_reason == "end_turn":
            answer = next((b.text for b in resp.content if b.type=="text"), "")
            log.info(f"[{run_id}] DONE in {step+1} steps | ${cost:.5f} | {time.time()-t0:.1f}s")
            return {"answer":answer,"trace":trace,"steps":step+1,"cost":round(cost,5)}

        results = []
        for b in resp.content:
            if b.type != "tool_use": continue
            out = tmap[b.name].run(**b.input) if b.name in tmap else f"ERROR: unknown tool {b.name}"
            trace.append({"step":step+1,"tool":b.name,"in":b.input,"out":out})
            log.info(f"[{run_id}] step {step+1} | {b.name}({b.input}) → {out[:80]}")
            results.append({"type":"tool_result","tool_use_id":b.id,"content":out,"is_error":out.startswith("ERROR")})

        msgs.append({"role":"user","content":results})

    return {"answer":"Max steps reached.","trace":trace,"steps":max_steps,"cost":round(cost,5)}
