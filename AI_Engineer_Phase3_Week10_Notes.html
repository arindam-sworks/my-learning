Three Memory Types

An agent that only knows what's in the current context window is profoundly limited. Real agents need three distinct memory systems — each serving a different time horizon and retrieval pattern.

Short-term
The conversation history. Everything said in this session. Lives in the messages array. Lost when the session ends. Limited by the context window.
Long-term
A vector store of documents, notes, and prior findings. Persists across sessions. Retrieved by semantic similarity. Scales to millions of chunks.
Episodic
A log of what the agent did in prior sessions — which questions it answered, what it found, what failed. Structured records, not raw text.

🧠 Why separate memory types?

Short-term memory (the messages array) is fast and always in context — but it grows with every turn, eventually overflowing the context window. Long-term memory scales infinitely but requires a retrieval step — you can't fit all your notes in every prompt. Episodic memory bridges both: it's a structured record of what the agent did, compact enough to always include but rich enough to prevent re-doing work. Real agents need all three, wired together.

Short-Term Memory

Short-term memory is the conversation history — the messages array you already know from Week 9. The challenge in Week 10 is managing it across many turns without overflowing the context window or losing important earlier context.

Python — Short-term memory manager
import tiktoken
from dataclasses import dataclass, field
from typing import List
import anthropic

enc    = tiktoken.get_encoding("cl100k_base")
client = anthropic.Anthropic()

def count_tokens(text: str) -> int:
    return len(enc.encode(str(text)))

@dataclass
class ShortTermMemory:
    max_tokens:   int  = 40_000   # keep ~40K tokens of history max
    messages:     list = field(default_factory=list)
    _token_count: int  = 0

    def add(self, role: str, content):
        """Add a message; evict oldest if over budget."""
        msg = {"role": role, "content": content}
        tok = count_tokens(str(content))
        self.messages.append(msg)
        self._token_count += tok
        self._evict_if_needed()

    def _evict_if_needed(self):
        """Remove oldest non-system messages until under budget.
        Always keep: the first user message (original task).
        Never evict: last 2 turns (immediate context).
        """
        while self._token_count > self.max_tokens and len(self.messages) > 4:
            # Remove index 1 (oldest non-first message)
            removed = self.messages.pop(1)
            self._token_count -= count_tokens(str(removed["content"]))

    def get(self) -> list:
        return self.messages.copy()

    def token_count(self) -> int:
        return self._token_count

    def summary(self) -> str:
        return (f"{len(self.messages)} messages | "
                f"~{self._token_count:,} tokens")

# Usage:
stm = ShortTermMemory(max_tokens=40_000)
stm.add("user", "What does chapter 3 say about transformers?")
# ... agent adds assistant responses, tool results ...
stm.add("user", "How does that relate to what chapter 5 says?")  # references prior turn
print(stm.summary())   # "4 messages | ~1,240 tokens"

Three eviction strategies

Python — Sliding window, summarise-and-compress, selective keep
def sliding_window(messages: list, keep_last_n: int = 10) -> list:
    """Simplest: keep only the N most recent messages.
    Fast but loses early context entirely."""
    if len(messages) <= keep_last_n:
        return messages
    # Always keep message[0] (original task)
    return [messages[0]] + messages[-keep_last_n+1:]


def summarise_and_compress(messages: list, keep_recent: int = 4) -> list:
    """Compress old messages into a summary, keep recent ones verbatim.
    More expensive (1 LLM call) but preserves meaning."""
    if len(messages) <= keep_recent + 2:
        return messages

    to_compress = messages[1:-keep_recent]  # everything except first + recent
    to_keep     = [messages[0]] + messages[-keep_recent:]

    conversation_text = "\n".join(
        f"{m['role'].upper()}: {str(m['content'])[:200]}"
        for m in to_compress
    )
    summary_resp = client.messages.create(
        model="claude-haiku-4-20250307",   # cheap model for summarisation
        max_tokens=512,
        messages=[{"role": "user", "content":
            f"Summarise this conversation history in bullet points. "
            f"Preserve all factual findings, tool results, and decisions made.\n\n"
            f"{conversation_text}"
        }]
    )
    summary = summary_resp.content[0].text

    # Inject summary as a system-like user message
    summary_msg = {
        "role": "user",
        "content": f"[CONVERSATION SUMMARY — earlier turns compressed]\n{summary}"
    }
    return [to_keep[0], summary_msg] + to_keep[1:]


def selective_keep(messages: list, important_keywords: list[str]) -> list:
    """Keep any message that contains important terms (findings, decisions).
    Drop purely conversational filler."""
    def is_important(msg) -> bool:
        content = str(msg.get("content", "")).lower()
        return any(kw.lower() in content for kw in important_keywords)

    kept = [messages[0]]   # always keep first
    for msg in messages[1:]:
        if is_important(msg):
            kept.append(msg)
    kept += messages[-4:]   # always keep last 4
    return list({id(m): m for m in kept}.values())  # deduplicate

Long-Term Memory

Long-term memory is the persistent vector store from Phase 2 (Weeks 7–8), now wired into the agent as a tool it can call. The key difference from a pure RAG system: the agent decides when and what to retrieve, rather than retrieving on every turn automatically.

Python — Long-term memory store
import chromadb, json
from chromadb.utils import embedding_functions
from datetime import datetime

class LongTermMemory:
    """
    Wraps ChromaDB as a semantic memory store.
    Stores both: document chunks (ingested at startup)
                 agent findings (written during runs).
    """
    def __init__(self, db_path: str = "./agent_memory"):
        self.client = chromadb.PersistentClient(path=db_path)
        ef = embedding_functions.OpenAIEmbeddingFunction(
            model_name="text-embedding-3-small"
        )
        # Two separate collections: documents vs agent knowledge
        self.docs     = self.client.get_or_create_collection("documents", embedding_function=ef)
        self.findings = self.client.get_or_create_collection("findings",  embedding_function=ef)

    # ── Document ingestion (run once at setup) ───────────────────
    def ingest_pdf(self, pdf_path: str):
        import PyPDF2
        reader = PyPDF2.PdfReader(pdf_path)
        fname  = pdf_path.split("/")[-1]
        chunks = []
        for page_num, page in enumerate(reader.pages):
            text = page.extract_text()
            if not text.strip(): continue
            # Simple paragraph chunking
            paragraphs = [p.strip() for p in text.split("\n\n") if len(p.strip()) > 100]
            for i, para in enumerate(paragraphs):
                chunks.append({
                    "id":   f"{fname}::p{page_num}::c{i}",
                    "text": para,
                    "meta": {"source": fname, "page": page_num + 1}
                })

        # Batch insert (max 100 per call)
        for i in range(0, len(chunks), 100):
            b = chunks[i:i+100]
            self.docs.add(
                documents=[c["text"] for c in b],
                ids=      [c["id"]   for c in b],
                metadatas=[c["meta"] for c in b],
            )
        print(f"Ingested {len(chunks)} chunks from {fname}")

    # ── Retrieval ────────────────────────────────────────────────
    def search_docs(self, query: str, k: int = 5) -> list[dict]:
        results = self.docs.query(
            query_texts=[query], n_results=k,
            include=["documents", "metadatas", "distances"]
        )
        return [
            {
                "text":   doc,
                "source": meta["source"],
                "page":   meta["page"],
                "score":  round(1 - dist, 3),
            }
            for doc, meta, dist in zip(
                results["documents"][0],
                results["metadatas"][0],
                results["distances"][0],
            )
        ]

    # ── Write agent findings to memory ───────────────────────────
    def store_finding(self, question: str, answer: str, sources: list[str]):
        """Agent writes discoveries back to long-term memory."""
        finding_id = f"finding::{datetime.utcnow().isoformat()}"
        self.findings.add(
            documents=[f"Q: {question}\nA: {answer}"],
            ids=      [finding_id],
            metadatas=[{"question": question, "sources": json.dumps(sources),
                        "timestamp": datetime.utcnow().isoformat()}],
        )

    def search_findings(self, query: str, k: int = 3) -> list[dict]:
        """Check if the agent already answered a similar question."""
        if self.findings.count() == 0: return []
        results = self.findings.query(query_texts=[query], n_results=k,
                                      include=["documents","metadatas","distances"])
        return [
            {"text": doc, "score": round(1-dist, 3),
             "timestamp": meta["timestamp"]}
            for doc, meta, dist in zip(
                results["documents"][0],
                results["metadatas"][0],
                results["distances"][0])
        ]
Two-collection architecture

Keep documents and findings in separate ChromaDB collections. Documents are immutable source material; findings are the agent's own conclusions. Separating them lets you search each independently — you can ask "have I already answered this?" before going back to the raw documents, saving both tokens and API calls.

Episodic Memory

Episodic memory is a structured log of what the agent did across sessions — which questions it answered, which documents it searched, what conclusions it reached, and what failed. It's compact enough to inject into every context window, preventing the agent from re-doing work it has already done.

Python — Episodic memory store
import json
from pathlib import Path
from datetime import datetime
from dataclasses import dataclass, asdict

@dataclass
class Episode:
    session_id:  str
    timestamp:   str
    question:    str
    answer:      str
    sources:     list[str]       # ["chapter3.pdf p.42", ...]
    tools_used:  list[str]       # ["search_docs", "search_docs"]
    steps:       int
    cost_usd:    float
    success:     bool

class EpisodicMemory:
    def __init__(self, path: str = "./episodic_memory.json"):
        self.path     = Path(path)
        self.episodes = self._load()

    def _load(self) -> list[Episode]:
        if self.path.exists():
            data = json.loads(self.path.read_text())
            return [Episode(**e) for e in data]
        return []

    def save_episode(self, ep: Episode):
        self.episodes.append(ep)
        self.path.write_text(json.dumps([asdict(e) for e in self.episodes], indent=2))

    def recent_context(self, n: int = 5) -> str:
        """Generate a compact summary of recent episodes to inject into system prompt."""
        if not self.episodes:
            return "No prior sessions."
        recent = self.episodes[-n:]
        lines  = ["Prior research sessions:"]
        for ep in recent:
            status = "✓" if ep.success else "✗"
            src    = ", ".join(ep.sources[:2])
            lines.append(
                f"  {status} [{ep.timestamp[:10]}] Q: {ep.question[:80]}...\n"
                f"     A: {ep.answer[:100]}...\n"
                f"     Sources: {src}"
            )
        return "\n".join(lines)

    def find_similar(self, question: str, threshold: float = 0.85) -> Episode | None:
        """Check if a near-identical question has been answered before.
        Simple: word overlap. For production: use embedding similarity."""
        q_words = set(question.lower().split())
        for ep in reversed(self.episodes):  # most recent first
            ep_words = set(ep.question.lower().split())
            overlap  = len(q_words & ep_words) / max(len(q_words | ep_words), 1)
            if overlap >= threshold:
                return ep
        return None
Episodic memory injection pattern

Always inject episodic context at the top of the system prompt, not as a user message. This frames it as background knowledge rather than a prior conversation turn. Keep it brief — the last 5 episodes in 3–4 lines each. If the agent answered a nearly identical question before (find_similar returns a hit), surface the cached answer immediately and skip retrieval entirely.

Context Window Management Across Turns

A multi-turn agent session accumulates tokens fast: conversation history + retrieved chunks + episodic summary + tool results + system prompt. You need a token budget manager that allocates space across all these components and never silently overflows.

Python — Multi-component context budget
@dataclass
class ContextBudget:
    # Claude Sonnet 4: 200K context, 8K max output
    model_context_limit: int = 200_000
    max_output:          int = 4_000

    # Fixed allocations
    system_tokens:    int = 800    # system prompt + episodic summary
    episodic_tokens:  int = 600    # recent episodes injected into system

    # Dynamic (computed per turn)
    history_tokens:   int = 0
    retrieved_tokens: int = 0

    @property
    def available_for_retrieval(self) -> int:
        used = (self.system_tokens + self.episodic_tokens +
                self.history_tokens + self.max_output)
        return max(0, self.model_context_limit - used)

    @property
    def utilization_pct(self) -> float:
        total_used = (self.system_tokens + self.episodic_tokens +
                      self.history_tokens + self.retrieved_tokens + self.max_output)
        return round(total_used / self.model_context_limit * 100, 1)

def build_context_for_turn(
    question:   str,
    stm:        ShortTermMemory,
    ltm:        LongTermMemory,
    epi:        EpisodicMemory,
    budget:     ContextBudget = None,
) -> dict:
    budget = budget or ContextBudget()

    # ── 1. Count current history ─────────────────────────────────
    history         = stm.get()
    budget.history_tokens = sum(count_tokens(str(m["content"])) for m in history)

    # ── 2. Retrieve docs within remaining budget ─────────────────
    retrieval_budget = budget.available_for_retrieval
    candidates       = ltm.search_docs(question, k=10)  # over-retrieve then trim
    selected_chunks  = []
    used_retrieval   = 0

    for chunk in candidates:
        chunk_tok = count_tokens(chunk["text"])
        if used_retrieval + chunk_tok > retrieval_budget: break
        selected_chunks.append(chunk)
        used_retrieval += chunk_tok

    budget.retrieved_tokens = used_retrieval

    # ── 3. Build episodic context for system prompt ───────────────
    episodic_ctx = epi.recent_context(n=5)

    print(f"Context: {budget.utilization_pct}% used | "
          f"{len(selected_chunks)} chunks | {budget.history_tokens} history tokens")

    return {
        "history":       history,
        "chunks":         selected_chunks,
        "episodic_ctx":   episodic_ctx,
        "budget":         budget,
    }
Total tokens = system + episodic_summary + history + retrieved_chunks + max_output
Must be ≤ model_context_limit (200K for Claude Sonnet 4)

Available for retrieval = limit − system − episodic − history − max_output

Memory Consolidation Strategies

Consolidation is the process of compressing or distilling memory to prevent unlimited growth while preserving the most valuable information. Three strategies for different scenarios.

Python — Three consolidation strategies
# ── Strategy 1: PERIODIC SUMMARISATION ───────────────────────
# Every N turns, compress the oldest half of history into a summary.
# Best for: long research sessions where all turns matter.

def maybe_consolidate(stm: ShortTermMemory, every_n_turns: int = 8) -> bool:
    """Consolidate when history exceeds trigger threshold."""
    if len(stm.messages) < every_n_turns * 2:
        return False
    # Summarise oldest half, replace with summary message
    condensed = summarise_and_compress(stm.messages, keep_recent=every_n_turns)
    stm.messages = condensed
    stm._token_count = sum(count_tokens(str(m["content"])) for m in condensed)
    return True

# ── Strategy 2: KNOWLEDGE DISTILLATION ───────────────────────
# At session end, extract structured facts from the conversation
# and write them to long-term memory as semantic knowledge.

def distill_session_to_ltm(
    stm:       ShortTermMemory,
    ltm:       LongTermMemory,
    session_id: str,
):
    """Extract key facts from the session and store as findings."""
    conversation = "\n".join(
        f"{m['role']}: {str(m['content'])[:300]}" for m in stm.messages
    )
    distill_resp = client.messages.create(
        model="claude-haiku-4-20250307",   # cheap model
        max_tokens=1024,
        messages=[{"role": "user", "content":
            f"""Extract 3-7 key factual findings from this research session.
Format as JSON list: [{{"fact": "...", "source": "...", "confidence": "high/medium/low"}}]
JSON only.

Conversation:
{conversation}"""
        }]
    )
    try:
        facts = json.loads(distill_resp.content[0].text)
        for fact in facts:
            ltm.store_finding(
                question=fact["fact"],
                answer=fact["fact"],
                sources=[fact.get("source", "session")],
            )
        print(f"Distilled {len(facts)} facts to long-term memory")
    except: pass  # distillation is best-effort


# ── Strategy 3: IMPORTANCE-WEIGHTED FORGETTING ───────────────
# Messages have an importance score. Low-importance messages
# are evicted first when space runs out.

def score_message_importance(msg: dict) -> float:
    content = str(msg.get("content", "")).lower()
    score   = 0.0
    # Tool results are usually important
    if "tool_result" in content or msg.get("role") == "tool":
        score += 2.0
    # Contains factual findings (numbers, names, dates)
    import re
    if re.search(r'\d{4}|\b[A-Z][a-z]+\s[A-Z]', str(msg.get("content", ""))):
        score += 1.5
    # Short filler messages (ok/thanks/understood) are low importance
    if len(content) < 50:
        score -= 1.0
    return score

RAG as an Agent Tool

The key architectural decision of Week 10: instead of always retrieving documents for every turn (pure RAG), we give the agent a search_documents tool it can call when it decides retrieval will help. This is more efficient and more intelligent — the agent can also check episodic memory first, and skip retrieval entirely for follow-up questions about facts it already found.

Python — RAG tools for the agent (3 tools)
from react_agent import Tool   # from Week 9

def make_rag_tools(ltm: LongTermMemory, epi: EpisodicMemory) -> list[Tool]:

    def _search_documents(query: str, k: int = 5) -> str:
        """Search the document knowledge base."""
        chunks = ltm.search_docs(query, k=k)
        if not chunks:
            return "No relevant documents found."
        parts = []
        for c in chunks:
            parts.append(
                f"[{c['source']}, page {c['page']} | relevance {c['score']}]\n{c['text']}"
            )
        return "\n\n---\n\n".join(parts)

    def _check_prior_research(query: str) -> str:
        """Check if this question or a similar one was answered before."""
        # Check episodic memory first (exact-ish match)
        prior = epi.find_similar(query, threshold=0.75)
        if prior:
            return (
                f"Found prior answer from {prior.timestamp[:10]}:\n"
                f"Q: {prior.question}\n"
                f"A: {prior.answer}\n"
                f"Sources: {', '.join(prior.sources)}"
            )
        # Check findings in vector store
        findings = ltm.search_findings(query, k=2)
        if findings and findings[0]["score"] > 0.85:
            return f"Found related prior finding (score {findings[0]['score']}):\n{findings[0]['text']}"
        return "No prior research found for this query."

    def _store_finding(question: str, answer: str, sources: str) -> str:
        """Store a new finding to long-term memory for future sessions."""
        source_list = [s.strip() for s in sources.split(",")]
        ltm.store_finding(question, answer, source_list)
        return f"Finding stored: '{question[:50]}...' → '{answer[:80]}...'"

    return [
        Tool(
            name="search_documents",
            description="Search the knowledge base of ingested documents. "
                        "Use when you need to find information in the source material. "
                        "Returns relevant passages with source citations.",
            schema={"type":"object","properties":{
                "query": {"type":"string","description":"Semantic search query"},
                "k":     {"type":"integer","description":"Number of chunks (default 5)","default":5}
            },"required":["query"]},
            fn=_search_documents
        ),
        Tool(
            name="check_prior_research",
            description="Check if this question was answered in a prior session. "
                        "ALWAYS call this before searching documents on a new question. "
                        "Avoids duplicating work across sessions.",
            schema={"type":"object","properties":{
                "query": {"type":"string","description":"The question to check"}
            },"required":["query"]},
            fn=_check_prior_research
        ),
        Tool(
            name="store_finding",
            description="Save an important finding to long-term memory. "
                        "Use after confirming a key fact from the documents. "
                        "This finding will be available in future sessions.",
            schema={"type":"object","properties":{
                "question": {"type":"string"},
                "answer":   {"type":"string"},
                "sources":  {"type":"string","description":"comma-separated source list"}
            },"required":["question","answer","sources"]},
            fn=_store_finding
        ),
    ]

Source Citations

Every answer the agent gives must be traceable to specific source chunks. Citations are what make a RAG agent trustworthy — and what interviewers look for when evaluating your RAG projects.

Python — Enforcing citations in the system prompt + parser
AGENT_SYSTEM = """You are a research assistant with access to a document knowledge base.

{episodic_context}

## Citation rules
After every factual claim from a document, you MUST cite the source
in this exact format: [source.pdf, p.N]

Example of correct citation:
"The transformer uses scaled dot-product attention [attention_paper.pdf, p.3].
This was first proposed in 2017 [attention_paper.pdf, p.1]."

If you cannot find the answer in the documents, say:
"I could not find this in the available documents."

## Workflow for each question
1. Call check_prior_research first — avoid repeating prior work
2. If no prior answer: call search_documents with a specific query
3. Synthesise from retrieved chunks with inline citations
4. Call store_finding to preserve important facts for future sessions
"""

import re
from dataclasses import dataclass

@dataclass
class Citation:
    source: str
    page:   int
    count:  int   # how many times cited

def extract_citations(text: str) -> list[Citation]:
    """Parse all [source.pdf, p.N] citations from answer text."""
    pattern = r'\[([^\]]+\.(?:pdf|md|txt)[^,\]]*),\s*p\.?(\d+)\]'
    matches = re.findall(pattern, text, re.IGNORECASE)

    citation_map: dict[tuple, int] = {}
    for src, page in matches:
        key = (src.strip(), int(page))
        citation_map[key] = citation_map.get(key, 0) + 1

    return [Citation(source=src, page=pg, count=cnt)
            for (src, pg), cnt in citation_map.items()]

def verify_citations(answer: str, retrieved_chunks: list[dict]) -> dict:
    """Check that cited sources actually appear in retrieved chunks."""
    citations     = extract_citations(answer)
    retrieved_srcs = {(c["source"], c["page"]) for c in retrieved_chunks}

    verified = []
    hallucinated = []
    for cit in citations:
        if (cit.source, cit.page) in retrieved_srcs:
            verified.append(cit)
        else:
            hallucinated.append(cit)

    return {
        "total":          len(citations),
        "verified":       len(verified),
        "hallucinated":   len(hallucinated),
        "citation_list":  citations,
        "is_grounded":    len(hallucinated) == 0,
    }
Always verify citation grounding

The model will occasionally cite sources that were not in the retrieved context — a hallucinated citation is more damaging than no citation, because it gives false confidence. After every answer, run verify_citations to check that every cited source actually appeared in the chunks you passed. Flag and log any hallucinated citations as reliability events.

Full System — RAG Agent with Memory

All components assembled: short-term memory, long-term memory, episodic memory, RAG tools, citations, and multi-turn conversation in a clean MemoryAgent class.

Python — memory_agent.py (complete)
class MemoryAgent:
    """
    RAG agent with three memory tiers.
    Remembers findings across multi-turn sessions.
    Every answer cites sources.
    """
    def __init__(self, db_path: str = "./agent_memory"):
        self.ltm       = LongTermMemory(db_path)
        self.epi       = EpisodicMemory(f"{db_path}/episodes.json")
        self.stm       = ShortTermMemory(max_tokens=40_000)
        self.rag_tools = make_rag_tools(self.ltm, self.epi)
        self.budget    = ContextBudget()
        self.session_id = str(uuid.uuid4())[:8]
        self._retrieved_this_turn: list[dict] = []

    def ingest(self, pdf_path: str):
        """Load a document into long-term memory."""
        self.ltm.ingest_pdf(pdf_path)

    def ask(self, question: str) -> dict:
        """Ask a question. Returns answer with citations and metadata."""
        self._retrieved_this_turn = []

        # Build system prompt with episodic context
        system = AGENT_SYSTEM.format(
            episodic_context=self.epi.recent_context(n=5)
        )

        # Add user question to short-term memory
        self.stm.add("user", question)
        maybe_consolidate(self.stm, every_n_turns=8)

        # Run the agent loop (from Week 9)
        result = self._run_loop(system)

        # Post-process: verify citations
        citation_check = verify_citations(result["answer"], self._retrieved_this_turn)

        # Write to short-term memory
        self.stm.add("assistant", result["answer"])

        # Save episode to episodic memory
        sources = [f"{c['source']} p.{c['page']}" for c in self._retrieved_this_turn[:3]]
        self.epi.save_episode(Episode(
            session_id=self.session_id,
            timestamp=datetime.utcnow().isoformat(),
            question=question,
            answer=result["answer"][:200],
            sources=sources,
            tools_used=[t["tool"] for t in result["trace"]],
            steps=result["steps"],
            cost_usd=result["cost"],
            success=citation_check["is_grounded"],
        ))

        return {
            "answer":          result["answer"],
            "citations":       citation_check["citation_list"],
            "grounded":        citation_check["is_grounded"],
            "steps":           result["steps"],
            "cost":            result["cost"],
            "history_tokens":  self.stm.token_count(),
        }

    def _run_loop(self, system: str) -> dict:
        """Agent loop: same as Week 9 but uses self.stm for history."""
        tmap     = {t.name: t for t in self.rag_tools}
        messages = self.stm.get()
        trace    = []
        cost     = 0.0

        for step in range(10):
            resp = client.messages.create(
                model="claude-sonnet-4-20250514",
                max_tokens=4096, system=system,
                tools=[t.api() for t in self.rag_tools],
                messages=messages,
            )
            cost += (resp.usage.input_tokens*3+resp.usage.output_tokens*15)/1e6
            messages.append({"role":"assistant","content":resp.content})

            if resp.stop_reason == "end_turn":
                answer = next((b.text for b in resp.content if b.type=="text"),"")
                return {"answer":answer,"trace":trace,"steps":step+1,"cost":round(cost,5)}

            results = []
            for b in resp.content:
                if b.type!="tool_use": continue
                out = tmap[b.name].run(**b.input) if b.name in tmap else f"ERROR: {b.name}"
                trace.append({"step":step+1,"tool":b.name,"in":b.input,"out":out[:200]})
                # Track retrieved chunks for citation verification
                if b.name == "search_documents":
                    self._retrieved_this_turn.extend(self.ltm.search_docs(b.input["query"]))
                results.append({"type":"tool_result","tool_use_id":b.id,"content":out})
            messages.append({"role":"user","content":results})

        return {"answer":"Max steps.","trace":trace,"steps":10,"cost":round(cost,5)}

    def end_session(self):
        """Distill session findings to long-term memory."""
        distill_session_to_ltm(self.stm, self.ltm, self.session_id)
        print(f"Session {self.session_id} ended. Memory consolidated.")

🧠 The architecture principle this week teaches

The fundamental insight of Week 10 is the difference between memory as context (short-term — just stuff in the prompt) and memory as retrieval (long-term — stuff you look up when needed). Every production AI system eventually faces this tension: you can't fit everything in context, but you also can't retrieve everything on demand. The three-tier architecture — STM for immediacy, LTM for scale, episodic for meta-cognition — is how you navigate it. This pattern recurs in every serious agent system you'll build in Phase 3.
